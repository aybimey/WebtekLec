HTTP FUNDAMENTALS

Introduction HTTP FUNDAMENTALS


    HTTP (Hypertext Transfer Protocol) is a protocol or rules used by the WWW (World Wide Web) when transferring data.
    Originally proposed by Tim Berners-Lee to communicate in the Web to help in the dissemination of information between institutes.
    HTTP Is a request-response protocol.
    Port 80 is the port that the server listens to or expects to receive from web client
    HTTP is based on client - server architecture
    HTTP provides support for other functionalities such as cache control, content media type(mime) specifications, language and character set\ specifications,content/transfer coding, client-server protocol negotitations,persistent connection, request pipelining, etc.

HTTP Versions


    HTTP 0.9 – This version was the first version of HTTP protocol and was defined in 1991. this version only consists of one method which is the GET.This method can only request a page from Server and has a response of an HTML Page.
    HTTP 1 - This version can be calle us a browser friendly protocol because it provides both request and response using http version number, status code and content type. This consist of 3 methods namely, GET, HEAD, POST.
    This is the commonly used version currently. This version supports the following methods: GET, HEAD, POST, PUT, DELETE, TRACE, OPTIONS.

HTTP Request & Response Message


Request Message


    Request Line - it begins to a method token that followed by the Request URI and the protocol version, and it ends to CRLF and it separated by space or SP character
    Message Header - It contains a general header and Entity header. The message header allows the client to pass the additional information about the request and also the client to the server, and acts like request modifiers.
    Empty Line - the empty line consists of CRLF that there’s no space
    Message Body - The http message used the entity body to connect to the request or the response, and it differs from entity body when the transfer-coding has been applied.

Response Message


    Status Line - The first line of a response message is the status line consists of protocol version or HTTP version followed by their corresponding numeric status code and its textual phrase, and it creates the specified properties.
    Message Headers - Allows the server to pass additional information about the response which cannot be placed in the status line, it gives information about th server and access to its resource requested by URI.
    Empty Line - In a response messages they use a generic message formatting for the transfer of entities consisting of start line and empty line
    Message Body - A message used to carry entity body associated with the request/response, message body only differs from entity-body if transfer-coding has been applied.

HTTP Request Methods


    Get – Get method requests and retrieves a representation of a specific resource.
    Head – Head method is similar to get request but it has no response body.
    Post – This method is used to send an entity to a specific resource. It usually triggers side effects.
    Put – This method changes current representations of the target resource with the request payload.
    Delete – This method will trigger deletion onto a specific resource.
    Options – This method describes the communication options for the specified target resource.
    Trace – Performs a message loop back test along the path towards the specified target resource.
    Connect – Method that creates a tunnel to the server defined by the target resource.

HTTP Status Code


    Information Status(1xx) - Informational status is a provisional response. The common status for informational is 100 which is Continue.
    Success Response(2xx) - This status shows if the response is successful. The common status is 200 which is OK. This means that the request succeeded.
    Redirection(3xx) - This status indicates that further action is needed to finished the request. The common example is the 302 which means Found. This means that the requested resource is temporary in the different URI.
    Client Error(4xx) - This status shows that the request that is send to the server has error or interrupted. The common example of this is the 403 Forbidden. This means that the server refused to give the resources that the client needs.
    Server Error(5xx) - This status indicates that the server is incapable or an error has occurred insending the response. The common example is the 502. This means that the server is getting invalid response from another server.
